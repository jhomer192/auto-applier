# Board pool for scripts/source.py — one `platform:token` per line.
#
# platform in {greenhouse, lever, ashby}. The token is the company's board slug:
#   greenhouse  job-boards.greenhouse.io/<token>
#   lever       jobs.lever.co/<token>
#   ashby       jobs.ashbyhq.com/<token>
#
# Every token here answered its board API with at least one open posting when it was
# added. A token that later dies just 404s, takes a `fails` tick in
# data/board_rotation.csv, and is auto-pruned after 4 consecutive misses.
# Add new companies here — never hardcode them in source.py.
#
# TIER MATTERS: Greenhouse + Lever are Tier 1 (they convert). Ashby is Tier 3
# HARD-AVOID per CLAUDE.md SITE ROUTING — it blocks at submit regardless of IP — so
# source.py mines greenhouse+lever only unless you pass --platforms explicitly. The
# Ashby tokens are kept for the rare role with no Tier-1 posting anywhere.
#
# There is no longer a separate "reserve" file. It used to hold boards that were live
# but had no matching job on the day they were checked. That split turned out to be
# false economy: mining 400 boards takes 6.6s at 16 workers, so a board costs
# essentially nothing to keep in rotation, and "no match today" says nothing about
# next week — postings turn over. Boards that stay barren are handled by the
# productivity weighting in pick_boards(), which demotes them after 3 fruitless mines
# without ever dropping them.
#
# Pool: 2097 boards (1553 greenhouse, 438 lever, 106 ashby).
